# Complete Web Design Business Platform - Replit Development Prompt

## PROJECT OVERVIEW
Build a full-stack web application for a web design business that consists of:
1. **Public-facing portfolio website** (for credibility/showcase)
2. **Client portal** (for invited clients to manage their projects)
3. **Admin dashboard** (for business owners to manage all clients)

**CRITICAL:** This is NOT a marketplace or application system. Clients are manually added by admins after offline conversations. The public site is purely for portfolio showcase.

---

## TECHNOLOGY STACK REQUIREMENTS

### Backend
- **Framework:** Node.js with Express.js
- **Database:** PostgreSQL (or MongoDB if preferred)
- **Authentication:** JWT tokens with bcrypt password hashing
- **Payment Processing:** Stripe API integration (Checkout Sessions)
- **File Storage:** Cloud storage (AWS S3, Cloudflare R2, or similar)
- **Email:** Nodemailer or similar for automated emails

### Frontend
- **Public Site:** React or Next.js (static generation preferred)
- **Client Portal:** React with protected routes
- **Admin Dashboard:** React with protected admin routes
- **Styling:** Tailwind CSS
- **Icons:** Lucide React or similar
- **Forms:** React Hook Form with validation

### Security Requirements
- HTTPS only (SSL certificate required)
- Password hashing with bcrypt (min 10 rounds)
- JWT tokens with expiration
- CSRF protection
- Rate limiting on API endpoints
- Input sanitization on all forms
- Prepared statements for database queries (SQL injection prevention)
- Environment variables for all secrets (.env file)

## DATABASE SCHEMA

### Users Table
```sql
users (
  id: UUID PRIMARY KEY,
  email: VARCHAR(255) UNIQUE NOT NULL,
  password_hash: VARCHAR(255) NOT NULL,
  role: ENUM('admin', 'client') NOT NULL,
  first_name: VARCHAR(100) NOT NULL,
  last_name: VARCHAR(100) NOT NULL,
  phone: VARCHAR(20),
  created_at: TIMESTAMP DEFAULT NOW(),
  updated_at: TIMESTAMP DEFAULT NOW(),
  last_login: TIMESTAMP,
  is_active: BOOLEAN DEFAULT TRUE,
  must_change_password: BOOLEAN DEFAULT TRUE
)
```

### Clients Table
```sql
clients (
  id: UUID PRIMARY KEY,
  user_id: UUID FOREIGN KEY REFERENCES users(id),
  
  -- Business Information
  business_legal_name: VARCHAR(255) NOT NULL,
  business_dba: VARCHAR(255),
  business_type: ENUM('sole_proprietor', 'llc', 'corporation', 'partnership', 'other'),
  industry: VARCHAR(100),
  tax_id: VARCHAR(20),
  business_phone: VARCHAR(20),
  business_email: VARCHAR(255),
  
  -- Business Address
  address_street: VARCHAR(255),
  address_city: VARCHAR(100),
  address_state: VARCHAR(50),
  address_zip: VARCHAR(10),
  address_country: VARCHAR(50) DEFAULT 'USA',
  
  -- Online Presence
  existing_website: VARCHAR(255),
  facebook_url: VARCHAR(255),
  instagram_url: VARCHAR(255),
  other_social: TEXT,
  
  -- Primary Contact
  primary_contact_title: VARCHAR(100),
  primary_contact_preferred_method: ENUM('email', 'phone', 'text', 'no_preference'),
  primary_contact_best_time: VARCHAR(100),
  
  -- Secondary Contact (Optional)
  secondary_contact_name: VARCHAR(200),
  secondary_contact_relationship: VARCHAR(100),
  secondary_contact_email: VARCHAR(255),
  secondary_contact_phone: VARCHAR(20),
  
  -- Project Assignment
  assigned_to: UUID FOREIGN KEY REFERENCES users(id),
  
  -- Client Source
  source: ENUM('referral', 'cold_call', 'social_media', 'previous_client', 'networking', 'other'),
  source_details: VARCHAR(255),
  
  -- Priority
  priority: ENUM('high', 'normal', 'low') DEFAULT 'normal',
  
  -- Internal Notes
  internal_notes: TEXT,
  
  -- Metadata
  created_at: TIMESTAMP DEFAULT NOW(),
  updated_at: TIMESTAMP DEFAULT NOW(),
  created_by: UUID FOREIGN KEY REFERENCES users(id)
)
```

### Projects Table
```sql
projects (
  id: UUID PRIMARY KEY,
  client_id: UUID FOREIGN KEY REFERENCES clients(id),
  
  -- Project Type
  project_type: ENUM('new_website', 'redesign', 'landing_page', 'ecommerce', 'other'),
  project_type_other: VARCHAR(255),
  
  -- Services (Stored as JSON array)
  services_included: JSON, -- Array of service codes
  
  -- Scope
  number_of_pages: INTEGER,
  special_requirements: TEXT,
  
  -- Pricing
  total_cost: DECIMAL(10,2) NOT NULL,
  payment_structure: ENUM('50_50', 'custom', 'full_upfront'),
  
  -- Timeline
  start_date: DATE,
  expected_completion_date: DATE,
  estimated_duration_weeks: INTEGER,
  actual_completion_date: DATE,
  
  -- Status
  status: ENUM('pending_payment', 'in_progress', 'design_review', 'development', 'client_review', 'revisions', 'completed', 'on_hold', 'cancelled') DEFAULT 'pending_payment',
  progress_percentage: INTEGER DEFAULT 0,
  
  -- Metadata
  created_at: TIMESTAMP DEFAULT NOW(),
  updated_at: TIMESTAMP DEFAULT NOW()
)
```

### Payments Table
```sql
payments (
  id: UUID PRIMARY KEY,
  project_id: UUID FOREIGN KEY REFERENCES projects(id),
  client_id: UUID FOREIGN KEY REFERENCES clients(id),
  
  -- Payment Details
  payment_number: INTEGER, -- 1st payment, 2nd payment, etc.
  amount: DECIMAL(10,2) NOT NULL,
  description: VARCHAR(255),
  due_date: DATE,
  
  -- Status
  status: ENUM('pending', 'paid', 'overdue', 'cancelled') DEFAULT 'pending',
  
  -- Stripe Integration
  stripe_payment_intent_id: VARCHAR(255),
  stripe_checkout_session_id: VARCHAR(255),
  stripe_transaction_id: VARCHAR(255),
  
  -- Payment Record
  paid_at: TIMESTAMP,
  paid_amount: DECIMAL(10,2),
  
  -- Invoice
  invoice_number: VARCHAR(50) UNIQUE,
  invoice_pdf_url: VARCHAR(500),
  
  -- Metadata
  created_at: TIMESTAMP DEFAULT NOW(),
  updated_at: TIMESTAMP DEFAULT NOW()
)
```

### Documents Table
```sql
documents (
  id: UUID PRIMARY KEY,
  client_id: UUID FOREIGN KEY REFERENCES clients(id),
  project_id: UUID FOREIGN KEY REFERENCES projects(id),
  
  -- Document Info
  document_type: ENUM('contract', 'invoice', 'deliverable', 'mockup', 'asset', 'other'),
  title: VARCHAR(255) NOT NULL,
  description: TEXT,
  
  -- File Storage
  file_url: VARCHAR(500) NOT NULL,
  file_name: VARCHAR(255) NOT NULL,
  file_size: INTEGER, -- in bytes
  file_type: VARCHAR(50), -- MIME type
  
  -- Signature (for contracts)
  requires_signature: BOOLEAN DEFAULT FALSE,
  is_signed: BOOLEAN DEFAULT FALSE,
  signature_data: TEXT, -- Base64 signature image or typed name
  signature_type: ENUM('drawn', 'typed'),
  signed_at: TIMESTAMP,
  signed_by_ip: VARCHAR(45),
  
  -- Visibility
  visible_to_client: BOOLEAN DEFAULT TRUE,
  
  -- Metadata
  uploaded_by: UUID FOREIGN KEY REFERENCES users(id),
  created_at: TIMESTAMP DEFAULT NOW(),
  updated_at: TIMESTAMP DEFAULT NOW()
)
```

## 2. AUTHENTICATION SYSTEM

### Login Page (`/portal/login`)
**Features:**
- Email and password fields
- "Remember me" checkbox (optional)
- "Forgot password?" link
- Login button
- Error messages for invalid credentials
- Redirect to appropriate dashboard based on role:
  - If role = 'admin' → `/admin/dashboard`
  - If role = 'client' → `/client/dashboard`

### Forgot Password Flow
1. User enters email
2. System sends password reset link (expires in 1 hour)
3. User clicks link, enters new password
4. Password updated, redirect to login

### First-Time Login (Clients Only)
- If `must_change_password = true`:
  - Force password change screen
  - Must create new password before accessing dashboard
  - Update `must_change_password = false` after change

### Session Management
- JWT tokens with 24-hour expiration
- Refresh token mechanism (optional but recommended)
- Auto-logout on token expiration
- "Stay logged in" option extends token to 7 days

6. Webhook updates payment status in database
7. Success page shows confirmation and receipt details
