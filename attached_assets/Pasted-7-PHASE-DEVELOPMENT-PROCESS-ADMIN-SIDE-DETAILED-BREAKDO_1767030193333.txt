PHASE 7: CLIENT REVIEW & DELIVERY

Status: client_review → revisions_complete → awaiting_final_payment → completed
Admin Action Required: Yes

Step 1: Client Reviews Site (Staging)

What Happens:

Client accesses staging site.

Client submits feedback/revision requests through the Duo portal.

Admin receives notification of submitted feedback.

Admin reviews and categorizes requests as:

Minor fixes (included): typos, spacing, small content or style adjustments

Major changes (out of scope): new pages, layout redesigns, new functionality

Bugs (covered under warranty): broken features or errors

Step 2: Handle Revision Requests

Admin responds to each request:

Accept & fix (minor / in scope)

Mark as out of scope (generate new quote if needed)

Request clarification

Admin applies approved changes on staging.

Admin notifies client:
“Revisions complete — please review again.”

Client reviews updated site.

Steps repeat until client is satisfied.

Step 3: Client Final Approval

Client clicks “Approve Final Website” in Duo.

System automatically:

Logs final approval timestamp

Locks design/development phases

Generates invoice for remaining 50% balance

Sends payment link

Updates status to awaiting_final_payment

Admin receives notification:
“Dan’s Electrical approved final site — awaiting final payment.”

Step 4: Final Payment Processing (Required Before Hosting)

Admin monitors payment status.

Once payment clears:

System marks payment as received

Status updates to payment_complete

Hosting setup phase becomes available
